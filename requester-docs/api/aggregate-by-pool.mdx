# Aggregate responses in a pool

<Announce/>

Starts aggregating responses to all completed tasks in the pool.

<Note mode="restriction" title="Restriction">

You can send a maximum of 5 requests of this kind per minute, 30 requests per hour, and 200 requests per day. Refer to the [Rate limiting](/docs/api/rate-limiting) section for the complete list of the request limitations in Toloka API.

</Note>

<Tabs tabs={[

]}/>

## Headers {#headers}

Title | Overview
----- | -----
**Authorization** | A token for account authorization. Add OAuth as a prefix. ||
**Content-Type** | Specifies the data format in the request body.

## Request body {#body}

```json
{
  "pool_id": "283",
  "type": "WEIGHTED_DYNAMIC_OVERLAP",
  "answer_weight_skill_id": "1289",
  "fields": [
    {
      "name": "result"
    }
  ]
}
```

Parameter | Overview
----- | -----
**pool_id** | **string**<br/><br/>Pool ID.
**type** | <AggregateByPoolTable1/>
**answer_weight_skill_id** | **string**<br/><br/>A skill that determines the weight of the Toloker's response.<br/><br/>Required if the `WEIGHTED_DYNAMIC_OVERLAP` aggregation type is selected.
**fields** | **object**<br/><br/>[Output data fields](/docs/guide/result-aggregation) to use for aggregating responses. For best results, each of these fields must have a limited number of response options.<br/><br/>If the `DAWID_SKENE` aggregation type is selected, you can only specify one value.
**fields.name** | **string**<br/><br/>The output data field name.

## Response {#response}

Contains operation details: Check the [operation status](/docs/api/operations). When it is completed [get the aggregation results](/docs/api/get-aggregated-result).

```json
{
  "id": "fadfe3jk-fdafue2-fda32890-fdafdi23",
  "type": "SOLUTION.AGGREGATE",
  "status": "PEDNDING",
  "submitted": "2018-22-10T14:18:35",
  "progress" : 0,
  "parameters": {
    "pool_id": "283"
  }
}
```

Parameter | Overview
----- | -----
**id** | **string**<br/><br/>Operation ID.
**type** | <AggregateByPoolTable2/>
**status** | <AggregateByPoolTable3/>
**submitted** | **string**<br/><br/>The UTC date and time the request was sent, in ISO 8601 format: `YYYY-MM-DDThh:mm:ss[.sss]`.
**progress** | **integer**<br/><br/>The percentage of the operation completed.
**parameters** | **object**<br/><br/>Parameters of the operation in the request.
