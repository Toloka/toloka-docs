# Get the operation log

<Announce/>

Gets the operation log.

You can get logs for asynchronous operations (`async_mode = true`). Such operations run in the background.

Logs are available only for the last month. To get the logs for an earlier period, contact [support](/docs/guide/troubleshooting/troubleshooting).

You can get the operation log:

- When you create [one or multiple](/docs/api/create-task) tasks.
- When you create [one or multiple](/docs/api/create-task-suite) task suites.
- [When issuing bonuses](/docs/api/create-bonus) to Tolokers.

If the operation was successful, the log contains `IDs` of the created objects, otherwise it contains the details of validation errors.

## Path parameters {#path-params}

Parameter | Overview
----- | -----
**operation_id** | Operation ID.

## Headers {#headers}

Title | Overview
----- | -----
**Authorization** | A token for account authorization. Add OAuth as a prefix.

## Response {#response}

Contains a JSON array describing each step of the operation.

```json
[
  {
    "type": <action type>,
    "success": <true/false>,
    "input": {
      <input values at the operation step>
    },
    "output": {
      <output values at the operation step or error details>
    }
  },
  ...
]
```

Parameter | Overview
----- | -----
**type** | <GetOperationLogTable1/>
**success** | <GetOperationLogTable2/>
**input** | **JSON**<br/><br/>The data in the `input` is copied from the input data passed when creating the operation.
**output** | <GetOperationLogTable3/>
