## Headers {#headers}

Title | Overview
----- | -----
**Authorization** | A token for account authorization. Add OAuth as a prefix.

## Query parameters {#query-params}

Specified in the link after the question mark; separated by `&`.

Parameter | Overview
----- | -----
**pool_id** | **string \| required**<br/><br/>ID of the pool to get tasks from.
**sort** | <GetTasksListTable1/>
**overlap** | **integer**<br/><br/>Tasks with an overlap equal to the specified value.
**Standard query parameters** | [limit](/docs/api/standard-query-parameters#limit), [id_gt](/docs/api/standard-query-parameters#id_gt), [id_gte](/docs/api/standard-query-parameters#id_gte), [id_lt](/docs/api/standard-query-parameters#id_lt), [id_lte](/docs/api/standard-query-parameters#id_lte), [created_gt](/docs/api/standard-query-parameters#created_gt), [created_gte](/docs/api/standard-query-parameters#created_gte), [created_lt](/docs/api/standard-query-parameters#created_lt), [created_lte](/docs/api/standard-query-parameters#created_lte), [overlap_gt](/docs/api/standard-query-parameters#overlap_gt), [overlap_gte](/docs/api/standard-query-parameters#overlap_gte) [overlap_lt](/docs/api/standard-query-parameters#overlap_lt), [overlap_lte](/docs/api/standard-query-parameters#overlap_lte).

## Query example {#request-example}

You can set up the display of the list of tasks in parts (for example, 10 tasks at a time):

1. Show the first 10 tasks, starting with the task with the lowest ID.
1. Show the remaining tasks (10 at a time) in ascending order.

## Response {#response}

Contains task data in the `items` array.

```json
{"items" : [{task 1}, {task 2}, ... {task n}], "has_more": true}
```
