# Working with data

To work with data, use the components with the `data.*` type. These include both core components and [special data components](/docs/template-builder/reference/datas).

## Core components

Core components are a part of other components which are used for accessing data from task files, Toloker responses, or internal data.

The core components include:

- `data.input`: The input data. For example, links to images that will be shown to Tolokers. In the Template Builder sandbox, you can set an example of input data.

- `data.output`: The output data. This is what you get when a Toloker clicks the **Submit** button.

- `data.internal`: The data available only from within the task. This data is not saved to the results. Use this data to calculate or store intermediate values.

- `data.local`: The local data available only from inside the component. This data is used in some auxiliary components, such as [helper.transform](/docs/template-builder/reference/helper.transform).

- `data.relative`: A special component for saving data. It's only available in the [field.list](/docs/template-builder/reference/field.list) component.

### Component properties `{#properties-core}`

The list of properties is the same for all core components.

**Title**|**Type**|**Overview**
----- | ----- | -----
`type`|<Term term="string"><StringType/></Term>|Name of the component.
`path`|<Term term="string"><StringType/></Term>|Path to the property containing data. Dots are used as separators: `path.to.some.element`<br/><br/>To specify the path to the array element, specify its sequence number starting from zero, for example: `items.0`
`default`|<Term term="any"><AnyType/></Term>|The value to use as the default data.<br/><br/>This value will be shown in the interface, so it might hide some placeholders, for example, in the [field.text](/docs/template-builder/reference/field.text) component.

## Special data components

Special data components send or return some data which is specific to Toloka and can be used, for example, to track some of the Toloker activities or the labeling progress.

The special data components include:

- [data.assignment-id](/docs/template-builder/reference/data.assignment-id): This component returns the assignment ID.

- [data.location](/docs/template-builder/reference/data.location): This component sends the device coordinates.

Refer to each component page for more details on component properties and usage examples.

### Component properties `{#properties-special}`

Special data components have only `type` property.

**Title**|**Type**|**Overview**
----- | ----- | -----
`type`|<Term term="string"><StringType/></Term>|Name of the component.
