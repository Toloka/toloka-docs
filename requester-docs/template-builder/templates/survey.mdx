# Survey

For this type of project, you can use the **Customer survey** preset. It has pre-configured validation and task layout.

<Button view="defaultInverse" size="l" type="link" url="https://ya.cc/t/ZbXJ-8Hb3ttFey">View example</Button>

<Cut title="Components used in the example">

- [field.radio-group](/docs/template-builder/reference/field.radio-group): For selecting one answer option.
- [field.checkbox-group](/docs/template-builder/reference/field.checkbox-group): For selecting one or more answer options.
- [field.text](/docs/template-builder/reference/field.text): For entering a single line of text.
- [condition.required](/docs/template-builder/reference/condition.required): Checks if at least one option is selected.
- [plugin.toloka](/docs/template-builder/reference/plugin.toloka): Customizes the task layout.

</Cut>

## What else can be configured `{#examples}`
- To let Tolokers enter long texts, replace the [field.textarea](/docs/template-builder/reference/field.textarea) component with [field.text](/docs/template-builder/reference/field.text).

  <Button view="defaultInverse" size="l" type="link" url="https://ya.cc/t/UgcP8H3C3ttFfQ">View example</Button>

- The [field.button-radio-group](/docs/template-builder/reference/field.button-radio-group) component adds buttons for selecting answers. It's better to use buttons if the question has 2–4 short answer options. If there are more answer options, it's better to use [field.radio-group](/docs/template-builder/reference/field.radio-group).

  <Button view="defaultInverse" size="l" type="link" url="https://ya.cc/t/uBROpMd23ttFfo">View example</Button>

If this preset doesn't meet your needs, see other examples in this section.
