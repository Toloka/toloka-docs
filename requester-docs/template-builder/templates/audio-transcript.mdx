# Audio transcription

For this type of project, you can use the **Transcribing audio recordings** preset. It has pre-configured validation, task layout, and shortcuts.

<Button themeColor="light" view="default" size="l" type="link" url="https://ya.cc/t/YrCIIps93ttEPe">View example</Button>

<Cut title="Components used in the example">

- [view.audio](/docs/template-builder/reference/view.audio): An audio player.
- [condition.played-fully](/docs/template-builder/reference/condition.played): Checks if a Toloker listened to the entire audio track.
- [field.button-radio-group](/docs/template-builder/reference/field.button-radio-group): Buttons for answer options.
- [field.textarea](/docs/template-builder/reference/field.textarea): Text input field.
- [condition.required](/docs/template-builder/reference/condition.required): Checks if the text field is filled in.
- [plugin.toloka](/docs/template-builder/reference/plugin.toloka): Customizes the task layout.
- [plugin.hotkeys](/docs/template-builder/reference/plugin.hotkeys): [Keyboard shortcuts](/docs/template-builder/best-practices/hotkeys).

</Cut>

## What else can be configured {#add-more}

- If you want to [check](/docs/template-builder/best-practices/conditions) whether the audio has been started, replace the [condition.played-fully](/docs/template-builder/reference/condition.played-fully) component with [condition.played](/docs/template-builder/reference/condition.played).

  <Button themeColor="light" view="default" size="l" type="link" url="https://ya.cc/t/fpVaBRrf3ttEQS">View example</Button>

- To put a short audio track on repeat, change the properties of the [view.audio](/docs/template-builder/reference/view.audio) component by adding `loop: true`.

  <Button themeColor="light" view="default" size="l" type="link" url="https://ya.cc/t/13Djihdu3ttERD">View example</Button>

If this preset doesn't meet your needs, see other examples in this section.

## Add pre-recognized text {#text}

  You can add text from input data to audio transcription tasks. For example, this is useful if an audio recording was recognized automatically, and now you need Tolokers to correct errors in the text. To do this, use the [view.text](/docs/template-builder/reference/view.text) component and refer to the number of the element in the input data array in the `content` property.
