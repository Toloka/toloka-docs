# Copywriting

For projects of this type, you can use the **Generating product descriptions** preset.

This preset helps to create a description of products based on the image and its name.

Take a look at the example: the labeling interface includes an image, text data, button and a text input area. If the Toloker clicks on the button, a search engine opens with a request about the product. Note that validation and task layout are already configured in this example.

## Set the desired text length {#text-length}

To set the desired text length, use the [condition.schema](/docs/template-builder/reference/condition.schema) component instead of [condition.required](/docs/template-builder/reference/condition.required). It won't let a Toloker submit a response if the text they entered is longer or shorter than specified.

<Cut title="Show code">

 ```json
 {
    "type": "condition.schema",
    "schema": {
      "type": "string",
      "minLength": 50,
      "maxLength": 200
    },
    "hint": "Please make sure your description contains 50 to 200 characters"
 }
 ```

</Cut>

## Add a layout {#add-layout}

To enhance Toloker's experience, you can highlight different types of data with colors using [view.alert](/docs/template-builder/reference/view.alert).

You can place it in the [view.list](/docs/template-builder/reference/view.list) along with the other components.

In this example, the description is highlighted with a blue border.

<Cut title="Show code">

```json
{
  "type": "view.alert",
  "theme": "info",
  "content": {
    "type": "view.text",
    "content": "Look at the product photo and write a brief description about it."
  }
}
```

</Cut>

## Use multiple columns {#several-columns}

To compare two texts, place them in adjacent columns using the [layout.columns](/docs/template-builder/reference/layout.columns) component. It allows you to set the column width and vertical alignment.

The [view.group](/docs/template-builder/reference/view.group) component adds frames around the texts to visually separate them from each other and make them easier to read.

The third column in the example contains two input fields for Tolokers to write their text.

 <Cut title="Show code">

 ```json
 {
    "type": "layout.columns",
    "fullHeight": true,
    "items": [
      {
        "type": "view.group",
        "label": {
          "type": "data.input",
          "path": "title1"
        },
        "content": {
          "type": "view.text",
          "content": {
            "type": "data.input",
            "path": "text1"
          }
        }
      },
      {
        "type": "view.group",
        "label": {
          "type": "data.input",
          "path": "title2"
        },
        "content": {
          "type": "view.text",
          "content": {
            "type": "data.input",
            "path": "text2"
          }
        }
      },
    ...
	]
 }
  ```

## Allow a Toloker to add input fields {#dynamic-field-add}

This is a complex example that consists of four main parts.

What's unique about this example is that a Toloker can add and remove text input fields. The condition is implemented using [field.list](/docs/template-builder/reference/field.list).

 <Cut title="Show code">

 ```json
 {
   "type": "field.list",
   "buttonLabel": "Add another name",
   "data": {
      "type": "data.output",
     "path": "names",
     "default": [
       ""
     ]
   },
   "render": {
     "type": "field.text",
     "placeholder": "Enter no more than one name here",
     "data": {
       "type": "data.relative"
     },
     "validation": {
       "type": "condition.any",
       "hint": "The name must be longer than three letters and include only letters and spaces",
       "conditions": [
         {
           "type": "condition.not",
           "condition": {
             "type": "condition.equals",
             "data": {
               "type": "data.output",
               "path": "result"
             },
             "to": "names_to_follow"
           }
         },
         {
           "type": "condition.schema",
           "schema": {
             "type": "string",
             "minLength": 3,
             "pattern": "^[А-Яа-яЁёa-zA-Z\\s]+<Formula withoutContainer={true} content="$"
           }
         }
       ]
     }
   },
   "validation": {
     "type": "condition.any",
     "hint": "enter at least one name",
     "conditions": [
       {
         "type": "condition.not",
         "condition": {
           "type": "condition.equals",
           "data": {
             "type": "data.output",
             "path": "result"
           },
           "to": "names_to_follow"
         }
       },
       {
         "type": "condition.not",
         "condition": {
           "type": "condition.empty",
           "data": {
             "type": "data.output",
             "path": "names"
           }
         }
       }
     ]
   }
 }
 ```

</Cut>

In addition, the example checks that each line has at least three characters, including letters and spaces. The condition is implemented using [condition.schema](/docs/template-builder/reference/condition.schema).

 <Cut title="Show code">

 ```json
 {
   "type": "condition.schema",
   "schema": {
      "type": "string",
      "minLength": 3,
      "pattern": "^[А-Яа-яЁёa-zA-Z\\s]+$"/>"
    }
 }
 ```

</Cut>

<Cut title="Components used in the example">

- [layout.columns](/docs/template-builder/reference/layout.columns): Places content in separate columns.
- [view.image](/docs/template-builder/reference/view.image): Displays an image.
- [data.input](/docs/template-builder/operations/work-with-data): The input data. For example, links to images that will be shown to Tolokers.
- [view.group](/docs/template-builder/reference/view.group): Groups components visually into framed blocks.
- [view.list](/docs/template-builder/reference/view.list): Displays data in a list.
- [field.radio-group](/docs/template-builder/reference/field.radio-group): Adds radio buttons for selecting an answer option.
- [data.output](/docs/template-builder/operations/work-with-data): The output data. This is what you get when a Toloker clicks the **Submit** button.
- [condition.required](/docs/template-builder/reference/condition.required): Checks that the data is filled in.
- [helper.if](/docs/template-builder/reference/helper.if): Allows to execute either one block of code or another, depending on the condition.
- [condition.equals](/docs/template-builder/reference/condition.equals): Checks whether the original value is equal to the specified value.
- [field.list](/docs/template-builder/reference/field.list): Allows a Toloker to add and remove new text input fields.
- [field.text](/docs/template-builder/reference/field.text): Adds a field for entering a short text.
- [condition.any](/docs/template-builder/reference/condition.any): Checks that at least one of the child conditions is met.
- [condition.not](/docs/template-builder/reference/condition.not): Returns the inverse of the specified condition.
- [condition.schema](/docs/template-builder/reference/condition.schema): Allows validating data using [JSON Schema](https://json-schema.org/learn/getting-started-step-by-step.html).
- [condition.empty](/docs/template-builder/reference/condition.empty): Checks that the data didn't get a value.
- [plugin.trigger](/docs/template-builder/reference/plugin.trigger): Triggers a specific action when an event occurs. The action is set in the `action` property.
- [plugin.hotkeys](/docs/template-builder/reference/plugin.hotkeys): Adds [keyboard shortcuts](/docs/template-builder/best-practices/hotkeys).
