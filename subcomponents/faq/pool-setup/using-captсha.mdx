<Cut title="Can I get more details on the best practices for using captchas? For which projects is it better to use captchas and how often?">

[Captcha](/docs/guide/captcha) is usually used in simple projects with automatic acceptance, like classification, categorization, or information search. These are cases where there are few response options and Tolokers don't need to upload files or write texts. It helps you filter out bots and sloppy Tolokers.

The frequency of issuing captchas is configured in the pool.

<Definition title="No">

Don't show captchas.

</Definition>

<Definition title="Low">

Show a captcha after every 20 assignments.

</Definition>

<Definition title="Average/High">

Show a captcha after every 10 assignments.

</Definition>

</Cut>
