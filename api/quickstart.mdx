# Overview

<Note mode="note" title="Note">

To better understand how Toloka works, first try setting up tasks in the Toloka interface using the [documentation](/docs-new/guide/main-steps) and then proceed to setting them up in the API.

</Note>

For example, create a project with tasks where Tolokers will determine the type of shoes.

The sample project described under "Quick start" is for the sandbox. To set up tasks in the Toloka production version, change the [resource URL](/docs-new/api/access#urls) and [OAuth token](/docs-new/api/access#token) in all requests.

## Steps to follow {#algorithm}

1. Post your tasks.

    - [Choose a platform and get a token](/docs-new/api/qs-placement#start).
    - [Create a project](/docs-new/api/qs-placement#project) (`POST /api/v1/projects`).
    - [Add a pool](/docs-new/api/qs-placement#pool) (`POST /api/v1/pools`).
    - [Upload the tasks](/docs-new/api/qs-placement#task) (`POST /api/v1/tasks`).
    - [Start (open) the pool](/docs-new/api/qs-placement#pool-run) (`POST /api/v1/pools/<pool_id>/open`).
    - [Check the tasks from the Toloker's interface](/docs-new/api/qs-placement#check).

1. Get the required results after the tasks are completed by the Tolokers.

    - [Get responses](/docs-new/api/qs-results) (`GET /api/v1/assignments?pool_id=<pool_id>`).
    - [Download attachments](/docs-new/api/get-attachment-list) (`GET /api/v1/attachments`).
