# Archive a project

<Announce/>

Sends the project to the archive.

If a project isn't being used, you can send it to the archive. To do this, all the pools in the project must be [archived](/docs-new/api/archive-pool). You can get the project ID from the [list of projects](/docs-new/api/get-prj-list).

## Response {#response}

The response contains information about the results of the operation.

```json
{
  "id" : "57068577e4b0bf7b07a0256f",
  "type" : "PROJECT.ARCHIVE",
  "status" : "SUCCESS",
  "submitted" : "2016-04-07T16:06:15.902",
  "started" : "2016-04-07T16:06:15.902",
  "finished" : "2016-04-07T16:06:15.902",
  "progress" : 100,
  "parameters" : {
    "project_id" : "1"
  }
}
```

Parameter | Overview
----- | -----
**id** | **string**<br/><br/>Operation ID.
**type** | <ArchivePrjTable1/>
**status** | <ArchivePrjTable2/>
**submitted** | **string**<br/><br/>The UTC date and time the request was sent, in ISO 8601 format: `YYYY-MM-DDThh:mm:ss[.sss]`.
**started** | **string**<br/><br/>The UTC date and time the operation started, in ISO 8601 format: `YYYY-MM-DDThh:mm:ss[.sss]`.
**finished** | **string**<br/><br/>The UTC date and time the operation was completed, in ISO 8601 format: `YYYY-MM-DDThh:mm:ss[.sss]`.
**progress** | **float**<br/><br/>The percentage of the operation completed.
**parameters.project_id** | **string**<br/><br/>Project ID.

You can check the operation status by sending requests to the `/operations` resource. For more information, see [Tracking operations](/docs-new/api/operations).
