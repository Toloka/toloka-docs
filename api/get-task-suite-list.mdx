# Get the list of task suites

<Announce/>

Gets the list of task suites in the pool.

## Headers {#headers}

Title | Overview
----- | -----
**Authorization** | A token for account authorization. Add OAuth as a prefix.

## Query parameters {#query-params}

Specified in the link after the question mark; separated by `&`.

Parameter | Overview
----- | -----
**pool_id** | **string**<br/><br/>ID of the pool to get task suites from.<br/><br/>You must specify either `task_id` or `pool_id`.
**task_id** | **string**<br/><br/>The task ID in the suites that were generated automatically using "smart mixing". You will get task suites that contain the specified task.<br/><br/>You must specify either `task_id` or `pool_id`.
**sort** | <GetTaskSuiteListTable1/>
**overlap** | **integer**<br/><br/>Suites with an overlap equal to the specified value.
**Standard query parameters** | [limit](./standard-query-parameters#limit), [id_gt](./standard-query-parameters#id_gt), [id_gte](./standard-query-parameters#id_gte), [id_lt](./standard-query-parameters#id_lt), [id_lte](./standard-query-parameters#id_lte), [created_gt](./standard-query-parameters#created_gt), [created_gte](./standard-query-parameters#created_gte), [created_lt](./standard-query-parameters#created_lt), [created_lte](./standard-query-parameters#created_lte), [overlap_gt](./standard-query-parameters#overlap_gt), [overlap_gte](./standard-query-parameters#overlap_gte) [overlap_lt](./standard-query-parameters#overlap_lt), [overlap_lte](./standard-query-parameters#overlap_lte).

## Query example {#request-example}

You can set up the display of the list of task suites in parts (for example, 10 task suites at a time):

1. Show the first 10 task suites, starting from the one with the lowest ID.
1. Show the remaining task suites (10 at a time) in ascending order.

## Response {#response}

Contains task suite data in the `items` array.

```json
{"items" : [{task suite 1}, {task suite 2}, ... {task suite n}], "has_more": true}
```
