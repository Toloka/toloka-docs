# Video classification

For this type of project, you can use the **Hand gesture classification** preset.

This preset helps you classify video content by specified categories. You can use it for content rating and moderation, detecting noise and other imperfections in the video, or training computer vision.

Take a look at the example: the labeling interface includes a video player, and several buttons for categories. Note that validation, keyboard shortcuts, and task layout are already configured in this Template Builder sample code.

## Check that the video is fully viewed

If you want to be sure that a Toloker watched the whole video, replace the [condition.played](/docs-new/template-builder/reference/condition.played) component with [condition.played-fully](/docs-new/template-builder/reference/condition.played-fully).

<Cut title="Show code">

  ```json
  {
    "type": "view.video",
    "validation": {
      "type": "condition.played-fully",
      "hint": "you need to watch the video to the end"
    },
    "url": {
      "type": "data.input",
      "path": "video"
    }
  }
  ```

</Cut>

## Add a layout {#add-layout}

To enhance Toloker's experience, you can highlight different types of data with colors using [view.alert](/docs-new/template-builder/reference/view.alert). You can place it in the [view.list](/docs-new/template-builder/reference/view.list) along with the other components.

In this example, the description is highlighted with a blue border, and the buttons are highlighted with a yellow one.

<Cut title="Show code">

```json
  {
    "type": "view.alert",
    "theme": "info",
    "content": {
      "type": "view.text",
      "content": "Watch the video and rate its quality."
    }
  },
  {
    "type": "view.alert",
    "theme": "warning",
    "content":{
      "type": "field.button-radio-group",
      "label": "rate the video quality",
      "options": [
        {
          "label": "Good",
          "value": "ok"
        },
        {
          "label": "Bad",
          "value": "bad"
        },
        {
          "label": "Failed to load",
          "value": "error"
        }
      ],
      "validation": {
      "type": "condition.required",
      "hint": "choose one of the options"
      },
      "data": {
        "type": "data.output",
        "path": "result"
      }
    }
  }
  ```

</Cut>

## Add clarifying questions to one of the options {#add-addition}

The [helper.if](/docs-new/template-builder/reference/helper.if) component displays an interface element after a specific response is selected. In this example, a Toloker can only select correct statements about the video if it is loaded.

  <Cut title="Show code">

```json
  {
    "type": "helper.if",
    "condition": {
      "type": "condition.equals",
      "data": {
        "type": "data.output",
        "path": "result"
      },
      "to": "yes"
    },
    "then": {
      "type": "field.checkbox-group",
      "label": "What's in this video?",
      "options": [
        {
          "label": "Planet Earth",
          "value": "earth"
        },
        {
          "label": "A planet rotating",
          "value": "turn"
        },
        {
          "label": "A blue planet",
          "value": "blue"
        }
      ],
      "data": {
        "type": "data.output",
        "path": ""
      },
      "validation": {
        "type": "condition.required",
        "hint": "Select one or more options"
      }
    }
  }
```

</Cut>

## Video and search query comparison {#search}

#### Does the video match the search query

Add a button that opens the search results and generate a search query link using the [helper.search-query](/docs-new/template-builder/reference/helper.search-query) component. To make sure that a Toloker clicked on the link and checked its contents, configure validation, as in the example.

<Cut title="Show code">

```json
{
  "type": "condition.link-opened",
  "hint": "Follow the link",
  "url": {
    "type": "helper.search-query",
    "query": {
      "type": "data.input",
      "path": "link"
    },
    "engine": "bing"
  }
}
```

</Cut>

<Button themeColor="light" view="default" size="l" type="link" url="https://ya.cc/t/u1WUKQIF3ttE3H">View example</Button>

#### Side-by-side video and web page comparison

You can display the web page in the built-in window using the [view.iframe](/docs-new/template-builder/reference/view.iframe) component. Place the video next to it using [layout.side-by-side](/docs-new/template-builder/reference/layout.side-by-side).

<Cut title="Show code">

```json
{
  "type": "view.iframe",
  "maxWidth": 400,
  "fullHeight": true,
  "url": {
    "type": "helper.search-query",
    "query": {
      "type": "data.input",
      "path": "link"
    },
    "engine": "bing"
  }
}
```

</Cut>

#### Side-by-side video and mobile web page comparison

This is a more complex example that compares the video with the results of a search query. The following components are added:

- [view.iframe](/docs-new/template-builder/reference/view.iframe): Displays the web page in an embedded window.
- [view.device-frame](/docs-new/template-builder/reference/view.device-frame): Wraps the window in a smartphone frame.
- [layout.side-by-side](/docs-new/template-builder/reference/layout.side-by-side): Places the video and the search results window next to each other.

<Cut title="Show code">

```json
{
  "type": "layout.side-by-side",
  "items": [
    {
      "type": "view.video",
      "content": {
        "type": "data.input",
        "path": "video"
      }
    },
    {
      "type": "view.device-frame",
      "maxWidth": 400,
      "content": {
        "type": "view.iframe",
        "maxWidth": 400,
        "fullHeight": true,
        "url": {
          "type": "helper.search-query",
          "query": {
            "type": "data.input",
            "path": "link"
          },
          "engine": "bing"
        }
      }
    }
  ]
}
```

</Cut>
