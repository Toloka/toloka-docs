# Working with data

To work with data, use the components with the type `data.*`:

- `data.input`: The input data. For example, links to images that will be shown to Tolokers. In the Template Builder sandbox, you can set an example of input data.
- `data.output`: The output data. This is what you get when you click the **Send** button.
- `data.internal`: The data available only from within the task. This data is not saved to the results. Use this data to calculate or store intermediate values.
- `data.local`: The local data available only from inside the component. This data is used in some auxiliary components, such as [helper.transform](/docs-new/template-builder/reference/helper.transform).
- `data.relative`: A special component for saving data. It's only available in the [field.list](/docs-new/template-builder/reference/field.list) component.

## List of properties {#properties}

The list of properties is the same for all components.

**Title**|**Type**|**Overview**
----- | ----- | -----
`type`|<WorkWithDataTable1/>|Name of the component.
`path`|<WorkWithDataTable2/>|Path to the property containing data. Dots are used as separators: `path.to.some.element`<br/><br/>To specify the path to the array element, specify its sequence number starting from zero, for example: `items.0`
`default`|<WorkWithDataTable3/>|The value to use as the default data.<br/><br/>This value will be shown in the interface, so it might hide some placeholders, for example, in the [field.text](/docs-new/template-builder/reference/field.text) component.
